Instrument Registration - Manual


Instrument registration is a process used by financial institutions (FIs), buyers, and sellers to register an instrument. In this tab, the user can either enter the instrument details manually or upload them. The instrument is then checked for validation and deviations before it can be financed.


Chirag Tated | 39 steps | 7 minutes


"Register an instrument by selecting 'Instrument Registration'."

2 To register a new instrument, select 'Instrument Registration'

If registering for multiple instruments, choose "Yes". Otherwise, select "No".

Open the dropdown to view the list of program names available on the platform.

Select the program name for instrument registration.

Choose the appropriate registration type from the available options. Manual registration involves entering instrument details individually. Upload registration allows the customer/FI to register all instruments using the provided Excel template. If they have chosen "Yes" for Multiple Instruments, the registration type will automatically be set to "Upload."

7 Select the appropriate "Limit Type" used for this instrument. Normal Limit Adhoc Limit

Select the appropriate "Process Type."
1.Invoice- This process type is generally used by buyer/seller to register the invoice in the corporate login screen.
2.Invoice & Finance- Financial institutions use this process type to initiate instrument registration on behalf of the buyer or seller and subsequently provide financing for the instrument.
3.Partial Finance- It is used in case the customer want to take multiple finances from the same instrument but at different time interval.

After entering all the mandatory details, click on the "CONTINUE" button

The next screen will present a overview of the program details

Click on the "+ Add Instrument" button to input instrument details.

Select the customer for whom the financial institution is registering the instrument.

The system will automatically populate the selected seller details on the screen based on the information entered in the buyer-seller details screen.

Select the "Instrument Reference Number*" field and enter the reference number of the instrument.

Select the appropriate date when the instrument was created.

Enter the amount of the instrument.

If the customer wants to repay any prior dues or charges from this finance, select the "Adjustment Amount" field and input the desired payment amount.

Based on the Finance % entered in the program details, the maximum finance amount for the instrument is automatically calculated.

Enter the finance amount for the customer. The amount should not exceed the maximum finance amount allowed.

The default tenure is automatically populated from the program setup screen. This value represents the maximum tenure days allowed. In the "Tenure" field, FI/Customer should enter the number of days, which can be equal to or less than the default tenure days.

The due date is automatically calculated when the "instrument date" and "tenure days" are entered.

The program setup screen automatically populates ROI, Adhoc ROI, and Grace Days details.

The Gross value represents the financing amount needed from the instrument. This value is automatically populated based on the finance amount entered.

The net value is automatically calculated by deducting the adjustment amount, interest, handling charges, and GST from the finance amount.

The beneficiary details of the seller are automatically populated from the buyer-seller details screen.

Once all the required information has been entered, click on the "Submit" button to proceed.

No. of instruments, Instrument value and finance amount data gets auto populated from the data we have added in "+Add Instrument" screen.

Select whether the loan booking branch is the same as the customer branch.
If "Yes", Loan booking branch tab becomes inactive.
If "No", Customer/FI have to select the branch of the customer's bank


Select the appropriate bank account details from which the disbursement should be made.

Choose the account details from the provided options.

Once all instrument details have been added, click on "VALIDATE." This action triggers a comprehensive validation process for the instruments, encompassing essential checks such as program limit, customer limit, and product limit.

A pop-up box will display comprehensive validation details, indicating whether the instrument is eligible for disbursement. If the instrument is successful, disbursement can proceed. In the case of an "error instrument," automatic rejection occurs upon approval.
1)Success Instrument- the instrument can be disbursed
2)Error Instrument- The instrument can't be disbursed. It will be rejected later on, when the checker approves the instrument.

To determine the cause of the error in the instrument, select "View" for the respective registered instrument.

In the "Failure Reason" tab, the displayed information will indicate the cause of the failure.

After validation, click "Check Deviation Level". This action will display any existing deviation levels caused by factors such as stale days or customer overdue.

If no deviation is present, the following message will be displayed on the screen

If a deviation level is present, the screen will display the corresponding deviation level details.

After enabling the "validate" and "Check deviation level" options, the Submit tab becomes active. If a deviation level is present and FI initiates submission, the instrument is escalated to the higher bank authority for approval. However, if no deviation is detected, the instrument is forwarded to the "checker" for approval.

To reject the instrument registration, navigate to the "Cancel" tab and provide the reason for rejecting the instrument. Then the instruments will move in the "Rejected" tab.
